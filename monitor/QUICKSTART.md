# Incident Monitor 快速指南

> 5 分钟上手 AI 事件监控

---

## 🚀 5 分钟快速开始

### 1. 安装（1 分钟）

```bash
cd monitor
npm install
```

### 2. 创建测试数据（1 分钟）

```bash
# 创建消息文件
cat > data/demo.jsonl << 'EOF'
{"ts": "1", "user": "user1", "text": "API 服务返回 500 错误！"}
{"ts": "2", "user": "user2", "text": "我也是，从 10 分钟前开始的"}
{"ts": "3", "user": "user1", "text": "影响支付功能，错误率 80%"}
EOF
```

### 3. 运行监控（1 分钟）

```bash
# 启动监控（每 5 秒检查一次）
node bin/monitor.js data/demo.jsonl --interval 5
```

输出：
```
🚨 Incident Monitor v1.0
============================================================
📁 监控目标：data/demo.jsonl
⏱️  轮询间隔：5 秒

按 Ctrl+C 停止监控

[Monitor] 启动监控：data/demo.jsonl
[Monitor] 发现 3 条新消息

============================================================
🔔 新消息提醒
============================================================
[2:40:01 PM] user2: 我也是，从 10 分钟前开始的
[2:40:02 PM] user1: 影响支付功能，错误率 80%
------------------------------------------------------------

📋 AI 分析 Prompt:
请分析以上事件讨论：
1. 识别问题类型和严重性 (P0/P1/P2/P3)
2. 列出缺失的关键信息
3. 给出排查步骤建议
4. 如需追问，生成最多 3 个问题
============================================================

💡 将以上内容发送给 AI 进行分析
```

### 4. 添加新消息（1 分钟）

打开另一个终端：

```bash
# 添加新消息
echo '{"ts": "4", "user": "user3", "text": "正在检查日志..."}' >> data/demo.jsonl
```

监控器会在 5 秒内检测到并提示你。

### 5. 发送给 AI 分析（1 分钟）

复制监控器输出的内容，发送给通义千问或 Claude：

```
请分析以下事件讨论：

[2:40:01 PM] user2: 我也是，从 10 分钟前开始的
[2:40:02 PM] user1: 影响支付功能，错误率 80%

1. 识别问题类型和严重性 (P0/P1/P2/P3)
2. 列出缺失的关键信息
3. 给出排查步骤建议
```

## 📋 常用命令

```bash
# 启动监控
node bin/monitor.js data/demo.jsonl

# 指定间隔（60 秒）
node bin/monitor.js data/demo.jsonl --interval 60

# 查看状态
node bin/monitor.js data/demo.jsonl --status

# 查看最近消息并生成 AI Prompt
node bin/monitor.js data/demo.jsonl --analyze

# 列出所有监控
node bin/monitor.js --list

# 停止监控
node bin/monitor.js data/demo.jsonl --stop
```

## 🔄 完整工作流程

```
1. 创建/导出 Slack thread 到文件
   ↓
2. 启动监控：node bin/monitor.js data/incident.jsonl
   ↓
3. 监控器定时读取文件（每 30 秒）
   ↓
4. 有新消息 → 显示 + AI Prompt → 复制发送给 AI
   ↓
5. 没有新消息 → 等待（不消耗 token）
   ↓
6. 检测到结束标记（如"已修复"）→ 自动停止
```

## 🤖 AI 分析 Prompt 模板

### 初始分析

```
请分析以下事件讨论：

{粘贴消息}

请：
1. 识别问题类型和严重性 (P0/P1/P2/P3)
2. 列出缺失的关键信息
3. 给出排查步骤建议
4. 如需追问，生成最多 3 个问题
```

### 进度更新

```
事件有新进展：

{粘贴新消息}

请：
1. 更新对问题的理解
2. 是否需要调整排查方向
3. 下一步建议
```

### 结束总结

```
事件已结束，这是完整讨论记录：

{粘贴所有消息}

请：
1. 总结根本原因
2. 列出处理过程中的关键决策
3. 给出预防建议
```

## 🎯 结束标记

以下关键词会自动停止监控：

- `已解决` / `已修复`
- `resolved` / `fixed` / `closed`
- `结束`
- `incident closed`
- `[CLOSED]`

## ❓ 常见问题

**Q: 如何停止监控？**
A: 按 Ctrl+C 或运行 `node bin/monitor.js <file> --stop`

**Q: 如何查看状态？**
A: 运行 `node bin/monitor.js <file> --status`

**Q: 如何跨会话继续？**
A: 状态自动保存，直接运行相同命令即可

**Q: 如何集成 Slack？**
A: 目前支持文件模式，Slack 集成开发中
