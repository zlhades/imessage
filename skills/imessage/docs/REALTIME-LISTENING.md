# iMessage 实时监听 - Qwen Code 集成

## 问题分析

当前的监听器是**后台独立进程**，和 Qwen Code 对话**没有连接**。

## 解决方案

### 方案 1: 在 Qwen Code 对话中直接轮询（简单）

在 Qwen Code 中说：

```
请每 5 秒检查一次 zlhades@icloud.com 的新消息，如果有新消息就执行并告诉我
```

Qwen 会：
1. 调用 MCP 工具读取消息
2. 检查是否有新消息
3. 执行并回复
4. 等待 5 秒，重复

**缺点**：Qwen Code 是对话式，不适合长时间轮询

---

### 方案 2: 使用 MCP 工具 + 手动触发（推荐）

配置好 MCP 后，在 Qwen Code 中：

```
查看并执行 zlhades@icloud.com 的最新消息
```

每次有新消息时，手动说这句话即可。

**优点**：简单可靠
**缺点**：需要手动触发

---

### 方案 3: 创建通知系统（高级）

监听器检测到新消息时：
1. 执行指令
2. 发送桌面通知
3. 写入共享文件
4. Qwen Code 监控共享文件变化

**实现复杂，但最接近"自动"**

---

### 方案 4: 快捷指令 + 本地服务器（最佳）

用手机快捷指令直接发送 HTTP 请求到本地服务器，绕过 iMessage 数据库。

**优点**：
- 实时
- 无需轮询
- 跨设备

---

## 当前推荐

使用**方案 2**：在 Qwen Code 中手动触发

```
执行 zlhades@icloud.com 的最后一条消息
```

或者创建一个快捷键：

```bash
alias imsg-exec='qwen -p "执行 zlhades@icloud.com 的最后一条消息" -y'
```

---

## 为什么后台监听器不够？

```
后台监听器进程
    ↓
调用 qwen -p（新会话）
    ↓
执行命令
    ↓
输出到日志文件 ❌（当前 Qwen 看不到）
```

**正确的架构应该是：**

```
Qwen Code 对话
    ↓
MCP 工具 (imessage_execute)
    ↓
直接读取并执行
    ↓
回复到当前对话 ✅
```
